## 三、数据库设计

### 表结构

| 表名 | 说明 | 关键字段 |
|------|------|----------|
| `t_lottery_result` | 开奖号码表 | `fissue`(期号,唯一), `fred1~6`, `fblue`, `fdraw_date`, `fball_key`(复合键去重,唯一索引) |
| `t_purchase_record` | 购买记录表 | `fissue`, `fred1~6`, `fblue`, `fball_key`(有索引), `fquantity`(注数), `fprize_level`, `fprize_money`, `fremark` |
| `t_predict_record` | 预测号码表 | `fissue`, `fred1~6`, `fblue`, `fball_key`(唯一索引), `fhit_red`, `fhit_blue`, `fprize_level` |

### 设计约定

- 所有表使用 `f` 前缀命名（如 `fid`, `fissue`, `fred1`）
- `fball_key` = `期号-红1-红2-红3-红4-红5-红6-蓝`（红球排序后拼接），用于去重
- 中奖等级：1=一等奖 ~ 6=六等奖，0=未中奖，NULL=待计算/待开奖

### 中奖规则

| 等级 | 命中条件 | 奖金 |
|------|----------|------|
| 一等奖 | 6 红 + 1 蓝 | 浮动 |
| 二等奖 | 6 红 + 0 蓝 | 浮动 |
| 三等奖 | 5 红 + 1 蓝 | 3,000 元 |
| 四等奖 | 5 红 + 0 蓝 或 4 红 + 1 蓝 | 200 元 |
| 五等奖 | 4 红 + 0 蓝 或 3 红 + 1 蓝 | 10 元 |
| 六等奖 | 任意红 + 1 蓝 | 5 元 |
| 未中奖 | 其他 | 0 元 |

## 五、号码推荐算法详解

### 算法流程

```
用户输入条件 → 参数校验 → 查缓存 → 未命中则全量计算 → 分页返回
```

1. **红球全量枚举**：递归回溯生成 C(33,6) = 1,107,568 种红球组合
2. **早期剪枝**：递归过程中同步应用剔除红球条件（跳过被剔除号码）
3. **多维过滤**：和值范围 / 区间比（低1-11、中12-22、高23-33）/ 奇偶比 / 剔除红球
4. **蓝球匹配**：对每个通过过滤的红球组合，遍历用户选择的蓝球（必选1~3个）
5. **上限截断**：结果超过 1000 组时截断并标记 `truncated=true`

### 过滤条件

| 条件 | 说明 | 输入格式 |
|------|------|----------|
| 和值范围 | 6个红球之和范围 | `sumMin=100, sumMax=110` |
| 区间比 | 低:中:高区号码数量 | `zoneRatio=2:2:2` |
| 奇偶比 | 奇数:偶数红球数量 | `oddEvenRatio=3:3` |
| 剔除红球 | 排除指定红球 | `excludeRed=[1,2,3]` |
| 选择蓝球 | 指定蓝球（必选1-3个） | `includeBlue=[5,8,12]` |

### 缓存策略

- 使用 Spring Cache + Caffeine
- 最多缓存 50 个不同查询条件，10分钟无访问后过期
- 缓存 key 由过滤条件（不含分页参数）序列化生成
- 缓存逻辑独立为 `RecommendCacheService`，避免自注入 hack，`@Cacheable` 通过正常代理调用生效

## 九、代码质量评估

### ✅ 做得好的地方

| 方面 | 评价 |
|------|------|
| **分层清晰** | Controller → Service → Mapper 分层规范，DTO/VO/Entity 职责分离 |
| **统一响应** | `Result<T>` + `PageResult<T>` 格式统一，错误码一致 |
| **全局异常处理** | `@RestControllerAdvice` 捕获 BizException / 校验异常 / 通用异常 |
| **参数校验** | DTO 使用 `@Valid` + `javax.validation` 注解，前端有二次校验 |
| **事务管理** | 关键写操作使用 `@Transactional(rollbackFor = Exception.class)` |
| **日志记录** | 使用 `@Slf4j`，关键操作有 info/debug 日志 |
| **构造器注入** | 大部分 Service 使用 `@RequiredArgsConstructor`（Lombok） |
| **中文注释** | 所有类、方法、关键逻辑都有中文注释 |
| **号码去重** | `fball_key` 复合键在插入前检查，DB 层有唯一索引兜底 |
| **缓存优化** | 推荐算法使用 Caffeine 缓存相同查询条件的结果 |

### 快速启动

```bash
start.bat
```

脚本自动完成：编译后端 → 启动后端（8080）→ 等待就绪 → 安装 serve → 启动前端（8081）→ 打开浏览器

**方式二：手动启动**

```bash
# 1. 初始化数据库
mysql -u root -p < dcb-backend/src/main/resources/sql/init.sql

# 2. 修改 application.yml 中的数据库密码

# 3. 启动后端
cd dcb-backend
mvn spring-boot:run

# 4. 启动前端（新终端）
cd dcb-frontend/app
npx serve -p 8081

# 5. 访问 http://localhost:8081
```

## 十二、总结

这是一个**完成度很高、代码规范**的双色球分析系统。两个需求规格共 21 个 Task 已全部实现，且额外扩展了批量操作、数据导出、期号自动补全等实用功能。

**技术亮点**：
- 后端采用标准 Spring Boot 三层架构，依赖注入规范，异常处理统一
- 推荐模块使用 C(33,6) 全量枚举 + 回溯剪枝 + Caffeine 缓存，兼顾准确性与性能
- 前端采用极简原生 JS + IIFE 模块化 + Hash 路由，零构建依赖
- `fball_key` 复合键设计优雅地解决了号码去重问题

**待改进**：数据库密码应外置到环境变量、`MAX_RESULT` 值需与需求一致、缓存逻辑可进一步解耦。
